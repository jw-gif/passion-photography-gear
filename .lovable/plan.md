# Branded Auth Email Templates

The email infrastructure and sender domain (`email.passionphotography.team`) are already configured. This plan adds custom-branded auth emails so messages like signup confirmations and password resets come from your domain and match the Passion Photography Team look.

## What gets created

Six React Email templates rendered through a server route that intercepts Supabase Auth events:

1. **Signup confirmation** — verify a new user's email
2. **Magic link** — passwordless sign-in
3. **Password recovery** — reset password
4. **Invite** — admin-invited new user
5. **Email change** — confirm a new email address
6. **Reauthentication** — sensitive action confirmation

Plus the `auth-email-hook` server route that Supabase calls to render and enqueue these emails through the existing queue.

## Brand styling

Pulled from `src/styles.css`:

- **Background:** white (`#ffffff`) email body — required for deliverability across all clients
- **Surface:** warm light gray card (`#F6F5F2`) for content
- **Text:** near-black (`#1A1A1A`) headings, muted gray (`#6B6B68`) body
- **Accent / CTA buttons:** bright cyan brand color (`#5BC7E8`) with dark text
- **Typography:** Inter (system fallback), display weight for the hero/title
- **Layout:** centered single column, ~560px max width, generous padding, rounded `12px` corners matching `--radius`
- **Header:** "Passion Photography Team" wordmark in dark cyan
- **Footer:** small muted text with sender address and a one-line tagline

## Files added

- `supabase/functions/auth-email-hook/index.ts` — webhook handler (auto-generated by scaffolding)
- `supabase/functions/auth-email-hook/deno.json`
- `supabase/functions/_shared/email-templates/*.tsx` — 6 brand-styled templates
- `supabase/config.toml` — function config block (auto-updated)

## Activation

Once scaffolded, the templates are deployed and Lovable Cloud activates them automatically as soon as DNS verification completes for `email.passionphotography.team`. You'll be able to monitor send activity and DNS status in **Cloud → Emails**.
